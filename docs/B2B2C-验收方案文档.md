## 1. 验收概述

### 1.1 验收目标

确保 B2B2C 商城系统满足需求文档、PRD 产品文档和技术方案文档中定义的所有功能需求和非功能性需求，系统可稳定运行在生产环境中。

### 1.2 验收范围

- **功能验收**：核心业务功能（商品、订单、支付、售后、营销、分销、商家入驻、管理后台）
- **性能验收**：响应时间、并发能力、吞吐量、资源利用率
- **安全验收**：数据安全、支付安全、接口安全、权限控制
- **兼容性验收**：多浏览器、多设备、多平台
- **可靠性验收**：系统可用性、故障恢复、数据一致性

### 1.3 验收原则

- **全面覆盖**：所有需求点必须有对应的验收用例
- **可重复**：验收用例可重复执行，结果一致
- **可度量**：验收标准可量化，不依赖主观判断
- **可追溯**：每个缺陷可追溯到需求点

### 4.2 压测工具

使用 k6 进行压测，示例脚本：

```javascript
// tests/performance/product-list.js
import http from 'k6/http';
import { check, sleep } from 'k6';

export const options = {
  stages: [
    { duration: '2m', target: 1000 },
    { duration: '5m', target: 5000 },
    { duration: '3m', target: 5000 },
    { duration: '2m', target: 0 },
  ],
  thresholds: {
    http_req_duration: ['p(99)<200'],
    http_req_failed: ['rate<0.01'],
  },
};

export default function () {
  const res = http.get('https://api.phpmall.com/shop/v1/products?category=1&page=1&per_page=20');
  check(res, {
    'status is 200': (r) => r.status === 200,
    'response time < 200ms': (r) => r.timings.duration < 200,
  });
  sleep(1);
}
```

## 5. 安全验收方案

### 5.1 安全测试清单

| 测试项 | 方法 | 通过标准 |
|--------|------|---------|
| SQL 注入 | SQLMap 扫描 | 无注入漏洞 |
| XSS 攻击 | 手动测试 + 自动化扫描 | 无 XSS 漏洞 |
| CSRF 攻击 | 手动测试 | Token 验证正确 |
| 接口鉴权 | 未授权访问测试 | 无权限无法访问 |
| 支付安全 | 篡改金额/重复请求 | 请求被拒绝 |
| 敏感数据 | 抓包分析 | 传输加密，存储加密 |
| 密码安全 | 弱密码测试 | bcrypt 加密，无法反解 |
| 文件上传 | 上传恶意文件 | 类型/大小限制有效 |
| 权限绕过 | 水平/垂直越权测试 | 无法越权访问 |
| 审计日志 | 关键操作检查 | 所有敏感操作留痕 |

### 5.2 安全扫描工具

- **静态扫描**：SonarQube（代码质量 + 安全漏洞）
- **依赖扫描**：Snyk（Composer/npm 依赖漏洞）
- **Web 扫描**：OWASP ZAP（接口漏洞扫描）
- **渗透测试**：手动渗透测试（登录、支付、权限）

